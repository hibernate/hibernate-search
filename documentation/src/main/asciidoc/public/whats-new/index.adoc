It also cleans up previously deprecated APIs.

== New features

[[stateless-session-support]]
=== Hibernate ORM `StatelessSession` support
Hibernate Search now supports Hibernate ORM's `StatelessSession` for both indexing and searching.
+
Indexing through a `StatelessSession` requires the
link:https://docs.hibernate.org/search/{hibernateSearchVersionShort}/reference/en-US/html_single/#coordination-outbox-polling[outbox-polling coordination strategy].
With outbox-polling, entities inserted, updated, or deleted through a `StatelessSession`
are indexed transparently, just like with a regular `Session`.
With the default coordination strategy, indexing through a `StatelessSession` will result in an exception.
+
Search queries can be executed from a `StatelessSession`
using the new `Search.session(StatelessSession)` entry point,
regardless of the coordination strategy.
+
See link:https://docs.hibernate.org/search/{hibernateSearchVersionShort}/reference/en-US/html_single/#limitations-stateless-session[limitations]
for details.

== Removals

[[v5migrationhelper-removal]]
Hibernate Search 5 migration helper::
The `hibernate-search-v5migrationhelper-engine` and `hibernate-search-v5migrationhelper-orm` artifacts are no longer published.
If you are still using the migration helper, you will need to migrate to the current Hibernate Search APIs.

[[jpa-orm-query-adapter-removal]]
JPA/Hibernate ORM query adapters::
Both JPA and Hibernate ORM query adapters alongside with `Search#toJpaQuery` and `Search#toOrmQuery` have been removed.
Use Hibernate Search query DSL API to work with your query and results.

== Dependency upgrades

[[orm-version]]
Hibernate ORM::
Hibernate Search targets the Hibernate ORM 8.0 series.
[[lucene-version]]
Lucene::
The Lucene-next backend is now based on Lucene 10.5.0.
[[opensearch-version]]
OpenSearch::
Hibernate Search is now also compatible with OpenSearch 3.7 as well as with previously compatible versions of OpenSearch.
